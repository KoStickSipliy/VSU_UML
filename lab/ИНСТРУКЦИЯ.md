# Система управления университетом

## Описание
Консольное приложение для управления структурой университета, включая факультеты, институты, сотрудников, курсы и проекты.

```powershell

```

## Функциональность

### Главное меню:
1. **Добавить факультет** - создание нового факультета с возможностью назначения декана
2. **Добавить институт к факультету** - добавление института к существующему факультету
3. **Добавить научного сотрудника** - регистрация научного работника
4. **Добавить административного сотрудника** - регистрация административного работника
5. **Добавить курс** - создание нового учебного курса
6. **Добавить проект** - создание нового исследовательского проекта
7. **Добавить участие в проекте** - назначение исследователя на проект
8. **Назначить исследователя в институт** - добавление исследователя в институт
9. **Назначить курс исследователю** - назначение курса для преподавания
10. **Показать информацию об университете** - отображение структуры университета
11. **Показать всех сотрудников** - список всех сотрудников с подробностями
12. **Показать все курсы** - список всех курсов
13. **Показать все проекты** - список всех проектов
14. **Показать статистику** - общая статистика по системе
0. **Выход** - завершение работы

## Реализованные классы

### Базовые классы:
- **Employee** (абстрактный) - базовый класс для всех сотрудников
- **SimpleDate** - класс для работы с датами

### Типы сотрудников:
- **ResearchAssociate** - научный сотрудник (может участвовать в проектах, вести курсы)
- **Dean** - декан (наследуется от ResearchAssociate, управляет факультетом)
- **AdministrativeEmployee** - административный сотрудник

### Структурные элементы:
- **University** - университет (содержит факультеты)
- **Faculty** - факультет (содержит институты и декана)
- **Institute** - институт (содержит исследователей)

### Учебная и научная деятельность:
- **Course** - учебный курс
- **Project** - исследовательский проект
- **Participation** - участие сотрудника в проекте

### Главный класс:
- **UniversitySystem** - консольный интерфейс для управления системой

## Особенности реализации

### Ввод-вывод:
- Каждый класс имеет методы `displayInfo()` или аналогичные для вывода информации
- Статические методы `readFromConsole()` для ввода данных с консоли
- Использование Scanner для чтения пользовательского ввода

### Связи между классами:
- Двунаправленные связи (например, между Institute и ResearchAssociate)
- Автоматическое обновление связей при добавлении элементов
- Коллекции для хранения связанных объектов

### Подсчёт сотрудников:
- Статический счётчик в классе Employee
- Автоматическое присвоение уникального ID каждому сотруднику
